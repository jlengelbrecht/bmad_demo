## What went well

**1. The per-story CR cadence held tight.**
Every story got an adversarial 3-reviewer pass (Blind Hunter / Edge Case Hunter / Acceptance Auditor) before the status closed to `done`. Patches landed in a follow-up commit, leaving review history visible. 13/23 (Epic 1) and 45/40 (Epic 2) patch-vs-defer ratios suggest reviews are surfacing real issues, not noise.

**2. The "rule of three from the start" call on `<AudienceCard>` aged well.**
Story 1.2's decision to extract the card on day one — instead of inlining three copies and refactoring later — meant Story 2.3's audience-entry rewrites had a stable component to lean on. Less rework, fewer diffs, no spec drift.

**3. Pre-existing baseline content was confronted, not papered over.**
Both Story 1.1 (scaffold merge) and Story 2.3 (audience-entry rewrite) inherited stale baseline content (an inconsistent README, a `00-start-here.md` that mentioned "create an account" against the no-auth architecture lock). Each story took the hit and replaced the lies. The shipped portal is internally consistent now.

**4. Test infrastructure landed early enough to bite.**
Playwright stood up after Story 1.3 with 6 tests; by Epic 2 close the test count was 60 unit + 16 e2e + lint + lint:links. Each new story extended the regression net rather than starting from scratch.

**5. `deferred-work.md` is doing real work as a backlog.**
22 items deferred across Epic 1 stories; 40 across Epic 2. Many already paired with the epic that owns the resolution (path-traversal hardening, contrast verification, capstone-glob review, etc.). This list will be Epic 5's punch-down list.

## What didn't go well

**1. AC literal text drifted from current tech-stack reality.**
Story 1.1's `tailwind.config.ts` AC was written against pre-v4 Tailwind. The implementation had to navigate a "literal AC vs current idiom" tension — resolved by generating a minimal config that v4 tolerates AND wiring it via `@config` in CSS. Story 2.5 had the same shape (architecture says `> 120 days`, epics says `≥ 120` — implementation sided with epics). The architecture document and epic AC text drifted from each other, and from the current ecosystem.

**2. Test-coverage gaps recurred at review.**
Multiple stories shipped with ACs that had no direct test (Story 2.1's dev-link-check plugin had no test; Story 2.2's keyboard-tab order had no test; Story 2.4's "no `Next:` on lesson 6" was unasserted). Reviewers caught them, but the pattern repeats — the implementation pass undertests behaviors the AC explicitly mandates.

**3. Hardcoded `localhost:3000` keeps surfacing.**
Story 1.3 and Story 2.2 reviews each flagged port-collision concerns. Next.js auto-falls-back to 3001+; tests and docs that hardcode :3000 silently break. Patched in 1.3 (port-collision note); test-side will need similar guarding when CI exists.

**4. Cross-platform skew (case sensitivity, path separators, fs symlinks) showed up at every link-checking helper.**
`dev-link-check` (Story 2.1), `loadContent` (Story 2.3), `check-links` (Story 2.4), `getLessonSequence` (Story 2.2) — all four needed the same pattern (decode + normalize + skip-symlinks + `path.resolve` containment). Each story's review re-discovered some piece of it. The first helper would have benefited from a shared utility.

**5. Robots `noindex/nofollow` toggling between stories.**
Story 1.2 review added `noindex` to the placeholder routes; Story 2.3 had to remove it because the routes graduated to real content. Test-side assertions had to flip with it. A "what's a placeholder?" framing decision early in Epic 1 would have prevented the round-trip.

## Patterns + lessons

| Pattern | Where it surfaced | Take-away |
|---|---|---|
| AC literal text drift from current ecosystem | 1.1 (Tailwind v4), 2.5 (threshold inequality) | Treat ACs as intent statements; if the literal contradicts current reality, document the deviation explicitly in Completion Notes |
| Coverage gap caught at review, not at impl | 2.1, 2.2, 2.4, 2.5 | Authoring stories: make AC6/test-coverage list verbatim, then assert each before closing the story |
| FS-helper pattern (decode + normalize + skip-symlinks + containment) reinvented per helper | 2.1, 2.3, 2.4 | Future fs helpers: lift to a shared utility OR explicitly document the parallel-evolution decision |
| Pre-existing baseline content needs cleanup | 1.1, 2.3 | When a story makes existing content user-visible, expect a baseline-hygiene sub-task |
| Placeholders accumulate temporary metadata | 1.2 → 2.3 (noindex toggle) | When introducing a placeholder, flag the lifecycle: "this metadata changes when the route graduates to real content" |
| Architecture doc and epics drifting from each other | 2.5 (threshold inequality) | Architecture doc owes a pass to reconcile with epics text |
