## Story

As the developer landing FR-3.24 / FR-6.7 / End-3 (HANDOFF.md generation),
I want `POST /api/capstone/handoff/generate` to write a `HANDOFF.md` inside the trainee's CHOSEN_DIR — summarizing what was produced, listing post-capstone team-adoption checklist items (CODEOWNERS placeholders, branch-protection, team-rituals checklist), and providing push-to-remote instructions,
So that the trainee leaves the capstone with both a working repo AND a one-pager their team can read on Monday morning that frames the next step (the brainstorm's "without this, trainees walk away with a directory; with it, they walk into Monday standup with a story" — line 215).

**AC1 — `POST /api/capstone/handoff/generate` Route Handler exists**
- File: `src/app/api/capstone/handoff/generate/route.ts`.
- Body (Zod): `{ sessionId: string }`.
- Behavior:
  1. Look up CHOSEN_DIR + tool + project name + outputFolder for the session.
  2. Verify the session is at Phase 9 (i.e., capstone-step `dev-story-1.1` is complete). Return 400 if not.
  3. Read the produced artifacts from CHOSEN_DIR to populate the template's "what was produced" section: the file paths and sizes of brief.md, prd.md, architecture.md, epics-and-stories.md, adr/0001-*.md, stories/1.1.md.

  5. Write the rendered content to `<chosenDir>/HANDOFF.md`. Overwrites if present (idempotent regeneration).
  6. Return `{ ok: true, path: '<chosenDir>/HANDOFF.md', sizeBytes: number }`.

**AC3 — Idempotency + overwrite safety**
- The handler overwrites `<chosenDir>/HANDOFF.md` if it exists. Trainees who want to re-generate (e.g., after updating their PRD post-capstone) can re-call.

- The path-traversal guard (writes inside CHOSEN_DIR only) is enforced via `path.resolve(chosenDir, 'HANDOFF.md').startsWith(chosenDir)` check. 500 on attempted traversal (defensive — should be unreachable).

**Why a static template instead of LLM-generated handoff content:**

The HANDOFF.md is a portal artifact, not an agent artifact. It needs to be deterministic (every trainee gets the same checklist), include accurate references (the portal's own paths + version), and avoid hallucinated guidance. Template + substitution is the cleanest shape. The trainee's AI tool already produced their bespoke artifacts (brief/PRD/etc.); the handoff is the orchestrator's contribution.

**Why atomic write:**

Trainees might trigger generation, then close the laptop, then re-open. A partial write on close interrupts the rendered file and a re-render would have to deal with malformed content. Atomic write (`tmp + rename`) is one line of code and removes the failure mode.
