## Findings outside the AC matrix (visual-smoke pass)

These are NOT AC violations — every AC is met as written. They are visible UX regressions that pass the AC literal text but fail the user experience:

| # | Finding | Affected stories | Severity |
|---|---|---|---|
| V1 | `prose` typography classes have no effect — `@tailwindcss/typography` plugin not installed | 2.2, 2.3 | HIGH |
| V2 | `rehype-autolink-headings` `#` content is visible to sighted users in heading text | 2.1, 2.2, 2.3 | HIGH |

Both filed as **R1** and **R2** in `implementation-readiness-report-2026-05-08.md`.

The fact that these slipped past the AC matrix is itself a finding: **the AC checklist works for behavioral correctness but not for UX correctness.** The Epic 1 retro's action item A2 (AC → test checklist as a story gate) should be expanded to AC → test → visual smoke for any story that ships rendered content.

The shipped code matches its AC contract — no false claims, no spec drift. Two real UX bugs slipped past because the test net is text-based; the visual smoke caught them.
